# Installing Plasma

## Requirements

- Git
- Bun `1.3.14`
- macOS, Linux, or Windows
- Foundry for local Anvil deployment
- a configured model provider for AI audit and fix generation

## Install a Published CLI

After a tagged CLI release exists:

```bash
curl -fsSL https://raw.githubusercontent.com/CYPHES-ATP/Plasma/main/install | bash
```

Install a specific release:

```bash
curl -fsSL https://raw.githubusercontent.com/CYPHES-ATP/Plasma/main/install |
  bash -s -- --version 1.0.0
```

Install a locally built binary:

```bash
./install --binary /absolute/path/to/plasma
```

The binary is stored at `~/.plasma/bin/plasma`. The installer adds only a
`plasma` shell alias; it never adds `~/.plasma/bin` to `PATH` and never creates
an `opencode` executable. Restart the terminal after installation or add the
alias manually:

```bash
alias plasma="$HOME/.plasma/bin/plasma"
```

## Build from Source

```bash
git clone https://github.com/CYPHES-ATP/Plasma.git
cd Plasma
bun install --frozen-lockfile
bun run build:cli
```

Find the binary:

```bash
find packages/opencode/dist -path '*/bin/plasma*' -type f
```

Run it:

```bash
./packages/opencode/dist/plasma-darwin-arm64/bin/plasma --help
```

Replace the target directory with the directory generated for your platform.

## Run from Source

Terminal interface:

```bash
bun run dev /path/to/solidity-project
```

Web development:

```bash
bun run dev:web
```

Desktop development:

```bash
bun run dev:desktop
```

## Build Desktop Artifacts

Signed production artifacts are generated by the protected GitHub release
workflow. For local development on macOS, build an unsigned preview:

```bash
OPENCODE_CHANNEL=prod bun --cwd packages/desktop build
OPENCODE_CHANNEL=prod bun --cwd packages/desktop package:mac:unsigned
```

`OPENCODE_CHANNEL` is an upstream compatibility variable. Production desktop
distribution also requires platform signing:

- Apple Developer ID certificate and notarization credentials
- Windows code-signing credentials
- protected GitHub release environment

Unsigned local packages are suitable for development, not public distribution.

When a signed build has been published, download it from:

```text
https://github.com/CYPHES-ATP/Plasma/releases
```

The desktop application contains its own server sidecar and uses isolated
desktop data, configuration, cache, state, database, authentication, and
loopback port settings. Open the app directly. Do not run `plasma serve` unless
you intentionally need a separate remote/API server.

Current preview artifacts are unsigned. On macOS, if Gatekeeper quarantines a
preview you built or downloaded for testing, remove quarantine explicitly:

```bash
xattr -dr com.apple.quarantine /Applications/Plasma.app
```

Production releases should use Apple notarization and Windows code signing
instead of requiring this preview-only step.

## Install Foundry and Anvil

Follow the official Foundry installation instructions, then verify:

```bash
forge --version
anvil --version
```

Start Anvil:

```bash
anvil
```

Plasma expects `http://127.0.0.1:8545` and chain ID `31337` by default.

## Configure a Model Provider

Use Plasma's provider settings or `/connect`. Do not place provider keys,
wallet secrets, or private keys in Solidity source files.

Legacy provider IDs and `opencode.json` configuration remain supported because
they are part of the upstream runtime contract.

For an authenticated remote server, set:

```bash
export PLASMA_SERVER_USERNAME=plasma
export PLASMA_SERVER_PASSWORD='use-a-secret-manager-or-secure-shell-secret'
plasma serve
```

This server command is independent of the desktop app.

Installer-managed builds can be upgraded with `plasma upgrade`. Plasma checks
the `CYPHES-ATP/Plasma` GitHub releases and runs the Plasma installer for the
selected version.

## Troubleshooting

Run verification:

```bash
bun run verify:plasma
```

Check the CLI:

```bash
plasma --version
plasma --help
```

If an older installer added `~/.plasma/bin` to `PATH`, remove that export from
your shell config and delete the obsolete compatibility executable:

```bash
rm -f "$HOME/.plasma/bin/opencode"
```

Check Anvil:

```bash
curl -s -X POST http://127.0.0.1:8545 \
  -H 'content-type: application/json' \
  --data '{"jsonrpc":"2.0","id":1,"method":"eth_chainId","params":[]}'
```
